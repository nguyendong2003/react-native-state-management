# 2. Redux Toolkit & Advanced (Redux Hiện đại và Nâng cao)

Học Redux cổ điển (Legacy Redux) giúp ta hiểu bản chất, nhưng trong thực tế phát triển dự án hiện nay, **Redux Toolkit (RTK)** là tiêu chuẩn bắt buộc phải sử dụng. Tài liệu này sẽ hướng dẫn bạn cách dùng Redux Toolkit để viết mã ngắn gọn, hiệu quả và các kiến thức nâng cao thường gặp trong React Native.

---

## 1. Tại sao lại là Redux Toolkit (RTK)?

Redux cổ điển thường bị phàn nàn bởi 3 vấn đề lớn:
1. Việc cấu hình Redux Store quá phức tạp và rườm rà.
2. Phải cài đặt thêm rất nhiều gói bổ sung (như `redux-thunk`, `reselect`, `immer`) để làm việc hiệu quả.
3. Đòi hỏi quá nhiều "boilerplate code" (viết đi viết lại file action types, action creators, reducers).

**Redux Toolkit (RTK)** ra đời nhằm giải quyết toàn bộ các vấn đề trên. Nó cung cấp các API được tối ưu sẵn, tích hợp sẵn Middleware hữu ích và tự động cấu hình Redux DevTools.

---

## 2. Các API cốt lõi trong Redux Toolkit

### A. `configureStore()`
Hàm thay thế cho `createStore` của Redux cũ. Nó tự động:
- Kết hợp các reducers con thành root reducer.
- Thêm các middleware mặc định (như `redux-thunk` để xử lý bất đồng bộ).
- Cấu hình sẵn kết nối với Redux DevTools.

```javascript
import { configureStore } from '@reduxjs/toolkit';
import counterReducer from './counterSlice';

const store = configureStore({
  reducer: {
    counter: counterReducer, // Đăng ký slice counter vào store
  },
});

export default store;
```

### B. `createSlice()`
Đây là API quan trọng nhất của RTK. Nó cho phép bạn khai báo State khởi tạo, các hàm Reducers và tự động sinh ra các Actions tương ứng tại **cùng một nơi**.

> [!TIP]
> **Immer Library Integration:** Bên trong `createSlice`, RTK sử dụng thư viện **Immer**. Điều này cho phép bạn viết mã thay đổi state trực tiếp giống như "mutating" (ví dụ: `state.value += 1`), nhưng Immer sẽ tự chuyển đổi nó thành cơ chế tạo bản sao bất biến (immutable update) ngầm bên dưới một cách an toàn.

```javascript
import { createSlice } from '@reduxjs/toolkit';

const initialState = {
  value: 0,
};

const counterSlice = createSlice({
  name: 'counter',
  initialState,
  reducers: {
    // Không cần dùng cấu trúc switch-case dài dòng nữa
    increment: (state) => {
      // Nhờ Immer, ta viết như thể đang thay đổi trực tiếp state cũ
      state.value += 1; 
    },
    decrement: (state) => {
      state.value -= 1;
    },
    incrementByAmount: (state, action) => {
      // Dữ liệu truyền vào action sẽ nằm trong action.payload
      state.value += action.payload;
    },
  },
});

// RTK tự động sinh ra các Action Creators tương ứng với reducers
export const { increment, decrement, incrementByAmount } = counterSlice.actions;

// Xuất file reducer để import vào store
export default counterSlice.reducer;
```

## 3. Xử lý Bất đồng bộ với `createAsyncThunk`

Trong lập trình web và mobile, phần lớn thời gian ta phải gọi API (bất đồng bộ). Trong Redux, các Reducer bắt buộc phải là Pure Function (đồng bộ). Do đó, ta phải dùng **Thunk** để xử lý bất đồng bộ trước khi gửi dữ liệu về Reducer.

`createAsyncThunk` nhận vào 2 đối số:
1. Một chuỗi đại diện cho tên action (ví dụ: `'users/fetchById'`).
2. Một hàm callback trả về một Promise (thường chứa logic gọi API).

Nó sẽ tự động sinh ra 3 action tương ứng với vòng đời của Promise:
- `pending`: Khi API bắt đầu được gọi (đang loading).
- `fulfilled`: Khi API trả về kết quả thành công (success).
- `rejected`: Khi API gặp lỗi (error).

```javascript
import { createSlice, createAsyncThunk } from '@reduxjs/toolkit';

// 1. Tạo Async Thunk để gọi API lấy thông tin User
export const fetchUserById = createAsyncThunk(
  'users/fetchById',
  async (userId, thunkAPI) => {
    const response = await fetch(`https://api.example.com/users/${userId}`);
    if (!response.ok) {
      throw new Error('Không thể lấy dữ liệu người dùng');
    }
    return await response.json(); // Đây sẽ là payload của action fulfilled
  }
);

// 2. Tạo Slice quản lý trạng thái API
const userSlice = createSlice({
  name: 'users',
  initialState: {
    data: null,
    loading: false,
    error: null,
  },
  reducers: {},
  // Xử lý các action sinh ra từ async thunk trong extraReducers
  extraReducers: (builder) => {
    builder
      .addCase(fetchUserById.pending, (state) => {
        state.loading = true;
        state.error = null;
      })
      .addCase(fetchUserById.fulfilled, (state, action) => {
        state.loading = false;
        state.data = action.payload; // Nhận dữ liệu API thành công
      })
      .addCase(fetchUserById.rejected, (state, action) => {
        state.loading = false;
        state.error = action.error.message; // Lưu thông báo lỗi
      });
  },
});

export default userSlice.reducer;
```

## 4. Quản lý Server State với RTK Query (RTKQ)

Nếu dự án của bạn sử dụng Redux Toolkit và cần gọi rất nhiều REST API, bạn nên sử dụng **RTK Query** thay vì tự viết `createAsyncThunk` thủ công. RTK Query là một công cụ mạnh mẽ đi kèm sẵn trong RTK để xử lý việc tải và lưu bộ nhớ đệm (caching) dữ liệu từ máy chủ.

Cách khai báo API Service bằng RTK Query:

```javascript
import { createApi, fetchBaseQuery } from '@reduxjs/toolkit/query/react';

// Định nghĩa một service sử dụng base URL và các endpoints
export const pokemonApi = createApi({
  reducerPath: 'pokemonApi',
  baseQuery: fetchBaseQuery({ baseUrl: 'https://pokeapi.co/api/v2/' }),
  endpoints: (builder) => ({
    getPokemonByName: builder.query({
      query: (name) => `pokemon/${name}`,
    }),
  }),
});

// RTK Query tự động sinh ra hook tương ứng cho mỗi endpoint
export const { useGetPokemonByNameQuery } = pokemonApi;
```

Sử dụng hook trong React Component:
```jsx
import React from 'react';
import { useGetPokemonByNameQuery } from './services/pokemon';

export default function PokemonInfo({ name }) {
  // Hook tự động gọi API, quản lý trạng thái loading, error và dữ liệu cache
  const { data, error, isLoading } = useGetPokemonByNameQuery(name);

  if (isLoading) return <Text>Đang tải dữ liệu...</Text>;
  if (error) return <Text>Đã xảy ra lỗi!</Text>;

  return <Text>Tên Pokemon: {data.name}</Text>;
}
```

## 5. Lưu trữ State lâu dài với `redux-persist` trong React Native

Trong các ứng dụng React Native di động, người dùng luôn mong muốn thông tin đăng nhập, giỏ hàng hoặc cấu hình ngôn ngữ vẫn còn được giữ lại sau khi họ tắt ứng dụng hẳn đi và mở lại.

Để thực hiện việc này, chúng ta kết hợp Redux với thư viện `redux-persist` và bộ nhớ thiết bị (`@react-native-async-storage/async-storage`).

### Quy trình thiết lập:

```javascript
import { configureStore, combineReducers } from '@reduxjs/toolkit';
import { persistStore, persistReducer } from 'redux-persist';
import AsyncStorage from '@react-native-async-storage/async-storage';
import authReducer from './authSlice';

// 1. Cấu hình persist
const persistConfig = {
  key: 'root',
  storage: AsyncStorage,
  whitelist: ['auth'], // Chỉ lưu trữ slice 'auth', các slice khác sẽ bị mất khi tắt ứng dụng
};

const rootReducer = combineReducers({
  auth: authReducer,
});

// 2. Tạo Persisted Reducer
const persistedReducer = persistReducer(persistConfig, rootReducer);

// 3. Khởi tạo Store với middleware tắt kiểm tra tuần tự (serializableCheck) của redux-persist
export const store = configureStore({
  reducer: persistedReducer,
  middleware: (getDefaultMiddleware) =>
    getDefaultMiddleware({
      serializableCheck: false, // Bắt buộc phải tắt cảnh báo này khi dùng redux-persist
    }),
});

export const persistor = persistStore(store);
```

Bao bọc ứng dụng bằng `PersistGate`:
```jsx
import { Provider } from 'react-redux';
import { PersistGate } from 'redux-persist/integration/react';
import { store, persistor } from './store';

export default function App() {
  return (
    <Provider store={store}>
      <PersistGate loading={<ActivityIndicator />} persistor={persistor}>
        <MainApp />
      </PersistGate>
    </Provider>
  );
}
```

## 6. Các Best Practices (Quy chuẩn thực tế) khi dùng Redux

### 1. Giữ cho State luôn phẳng (State Normalization)
Tránh lồng ghép các cấu trúc object quá sâu (ví dụ: `state.users.posts.comments`). Hãy cố gắng lưu trữ dữ liệu dưới dạng "phẳng" (dùng ID làm key tham chiếu giống như database) để cập nhật và truy xuất tối ưu hiệu năng nhất.

### 2. Sử dụng Selector thông minh (Memoized Selectors)
Khi truy xuất dữ liệu từ store, nếu bạn phải tính toán phức tạp (như lọc danh sách, sắp xếp dữ liệu), hãy sử dụng `createSelector` (được tích hợp sẵn trong RTK). Selector này sẽ cache lại kết quả và chỉ tính toán lại khi các trường state phụ thuộc thực sự thay đổi, ngăn component bị re-render vô ích.

```javascript
import { createSelector } from '@reduxjs/toolkit';

const selectTodos = (state) => state.todos.items;
const selectFilter = (state) => state.todos.filter;

// Chỉ tính toán lọc danh sách khi `items` hoặc `filter` thay đổi
export const selectVisibleTodos = createSelector(
  [selectTodos, selectFilter],
  (todos, filter) => {
    switch (filter) {
      case 'completed':
        return todos.filter(t => t.completed);
      default:
        return todos;
    }
  }
);
```

### 3. Không lưu các dữ liệu không tuần tự hóa (Non-serializable Data)
Không bao giờ được lưu các giá trị như: **Functions (hàm)**, **Promises**, hoặc **đối tượng Class instances** vào Redux Store. Chỉ lưu trữ các dữ liệu JSON thuần túy (plain JS objects, arrays, strings, numbers, booleans).
